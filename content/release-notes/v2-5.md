Platform release 2.5 shipped to all hubs the week of June 1, 2026.

## Adaptive scheduling is GA

Adaptive watering leaves beta. Changes since the beta:

- **Skip records.** Declined evaluations are now first-class run-history
  entries with explicit reasons (`skipped:humidity_above_target`,
  `skipped:deferred_to_lit_hours`), ending the "silent gap" diagnosis problem.
- **`next_planned_run`** is exposed in the schedule API so the engine's
  current intention is inspectable.
- **Evaporation-curve learning** is bounded per zone and resets when a zone's
  profile changes.

Existing beta schedules migrate in place; guardrail parameters are unchanged.

## Webhook signatures v2

New subscriptions sign deliveries with HMAC-SHA256 over the raw body
(`X-Verdant-Signature`). Existing v1 subscriptions (SHA-1) continue to work
but are deprecated — rotate by deleting and recreating the subscription, which
issues a fresh signing secret. v1 signing stops in release 2.7.

## Pump flow alerts

Runs that complete with zero measured flow now raise a hardware alert and a
`watering` event instead of silently "succeeding." This catches clogged
nozzles and lost pump prime the day they happen.

## Smaller changes

- Manual runs now appear in run history with reason `manual`, consistent with
  engine runs.
- `GET /v2/webhooks` reports each subscription's 7-day delivery success rate.
- Calibration reminders fire two weeks before due, up from one.

## Deprecations

- Webhook v1 (SHA-1) signatures — removal in 2.7.
